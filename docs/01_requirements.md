# 01. Requirements

## 1. 프로젝트 개요

### 프로젝트명
ShopSphere – 중소 브랜드를 위한 커머스 플랫폼

### 프로젝트 목적
소규모 브랜드가 별도 개발 없이 온라인 쇼핑몰을 운영할 수 있도록 지원하는 커머스 플랫폼을 구축한다.

### 타겟 사용자
- 판매자 (Seller)
- 일반 고객 (Customer)
- 관리자 (Admin)

### 핵심 가치
- 주문 데이터의 정합성 보장
- 재고 동시성 문제 해결
- 확장 가능한 아키텍처 설계

### 예상 트래픽
- 일 평균 방문자: 5,000명
- 최대 동시 주문: 300건

---

## 2. 유저 스토리

| 사용자 | 시나리오 | 기대 결과 |
|--------|----------|------------|
| 고객 | 상품을 장바구니에 담는다 | 장바구니에 상품이 저장된다 |
| 고객 | 주문을 완료한다 | 재고가 감소하고 주문이 생성된다 |
| 고객 | 결제 실패 | 주문 상태가 CANCEL로 변경된다 |
| 판매자 | 상품을 등록한다 | 상품이 목록에 노출된다 |
| 관리자 | 비정상 주문을 취소한다 | 재고가 복구된다 |

---

## 3. 기능 명세

### 3-1. 필수 기능 (MVP)

- 회원 가입 / 로그인
- 상품 조회
- 장바구니 기능
- 주문 / 결제
- 재고 관리

### 3-2. 확장 기능

- 리뷰 기능
- 할인 쿠폰 기능
- 관리자 대시보드
- 주문 통계

### 기능 우선순위 표

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 회원 가입 | 이메일 기반 회원 가입 | High |
| 로그인 | JWT 인증 | High |
| 상품 조회 | 카테고리별 조회 | High |
| 주문 생성 | 재고 감소 + 주문 생성 | Critical |
| 리뷰 작성 | 구매자만 작성 가능 | Medium |

## 4. 기술 스택 선정 이유

### Backend
**Spring Boot**
- 빠른 개발 환경 제공
- 의존성 관리 용이
- 대규모 커뮤니티 지원

**JPA**
- 객체 중심 도메인 설계 가능
- 영속성 컨텍스트 기반 1차 캐시 제공
- 트랜잭션 관리에 유리

---

### Database
**MySQL**
- ACID 보장
- 트랜잭션 기반 RDB
- 커머스 도메인에 적합

---

### Cache
**Redis**
- 재고 동시성 제어를 위한 분산 락 활용
- 상품 목록 캐싱

---

### Authentication
**JWT**
- Stateless 인증 방식
- 서버 확장에 유리
